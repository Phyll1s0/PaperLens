# 如何用 PaperLens 精读一篇论文

这篇文章只讲“怎么用”。如果你已经能打开 PaperLens，但不知道下一步该点什么，按这篇走就行。

## 1. 先把模型通道跑通

打开 PaperLens 后，先看左侧“模型”区域。新用户建议优先选这几个入口：

- `Kimi Code Direct`：使用 Kimi Code Console 生成的 `sk-kimi-` Key，不需要 Claude CLI。
- `DeepSeek`：使用 DeepSeek 的 OpenAI-compatible API。
- `OpenAI`：使用 OpenAI API。
- `自定义`：接其他兼容 `/chat/completions` 的服务。

如果你没有安装 Claude Code，先不要选 `Claude Code 本机配置`。这个入口只适合已经在本机配置好 Claude Code CLI 的用户。

配置时按顺序做：

1. 选择 Provider。
2. 检查 Base URL 和 Model。
3. 粘贴完整 API Key。控制台列表里 `sk-...` 的脱敏值不能用。
4. 如果网络需要代理，填写 Proxy URL。
5. 点击“测试连接”。

测试成功后再上传论文。这样出问题时能更快判断是模型通道问题，还是 PDF 解析问题。

## 2. 上传 PDF

在左侧 PDF 区选择文件。第一次使用时建议保持：

- `AI 分段`：开启。
- `上传后自动翻译讲解`：开启。
- 模式：先用 `精读`。

`精读` 会更重视全文结构、章节上下文、跨页段落和图表关系。`快速` 更适合粗看论文，不适合你要认真做笔记的情况。

点击“上传并解析”后，PaperLens 会先提取 PDF，再做结构规划、分段和逐段讲解。长任务会进入后端队列；刷新页面后仍能同步进度。

## 3. 读主界面

论文加载后，主区域按段落显示。每个段落通常包含：

- 原文。
- 翻译。
- 讲解。
- 关键词或术语。
- 相关图片、表格、公式、代码块。
- 单段追问入口。

如果某段图片很多，或者公式/代码识别不准，先不要急着全量重跑。优先打开“质量报告”看它到底是哪类问题。

普通文本型 PDF 不要求你先安装 Poppler 或 OCR 工具。它们是增强项：Poppler 改善页面图片和裁剪，OCRmyPDF/Tesseract 用于扫描版 PDF。为什么不默认完全交给 AI 解析，见 [PDF 解析策略](PDF_STRATEGY.md)。

## 4. 看质量报告

“质量报告”是排查入口。它会汇总：

- 分段数量和隐藏段落。
- 失败、未完成或可补跑段落。
- 图片、表格、公式、代码裁剪质量。
- 导出风险。
- Provider 和任务状态。

常见处理方式：

- 只有少数段落失败：点“补跑未完成”。
- 单段讲解不好：在该段上单独重跑。
- 分段整体错：用“分段调试”确认后，再“重分段+全跑”。
- 图片/公式裁剪不好：看视觉 QA 或图表卡片里的裁剪质量，再决定是否重建视觉结构。

## 5. 问单段问题

每段都可以追问。适合问：

- 这个公式里的变量分别是什么意思？
- 这段和上一段有什么关系？
- 这个图为什么支持作者的结论？
- 这段是不是在讲方法、实验还是局限？

段落问答会结合附近段落、章节上下文和相关图表 caption，不只是孤立看当前句子。

## 6. 修复分段

分段不对时，先分清问题大小。

小问题：

- 两段应该合并。
- 一段应该拆开。
- 某个标题、链接、会议名被当成正文。
- 某个公式或资源链接被隐藏了但应该恢复。

这类问题优先用“本地修复”“分段调试”或段落级操作，不要立刻重跑整篇。

大问题：

- 正文起点错了。
- References 被当成正文。
- 后半篇大量漏段。
- 双栏跨页段落连续错。

这类问题再考虑“重分段+全跑”。它会重新分段并清空旧分析结果，所以成本更高。

## 7. 处理图片、公式和代码

PaperLens 会把图片、表格、公式和代码尽量做成可点击的结构块。你可以：

- 点击图片或公式查看大图。
- 用“定位”回到原 PDF 页面位置。
- 用“裁剪”或视觉 QA 检查边界。
- 用“查看/下载”拿到对应裁剪图。

如果外面预览空白但点开正常，通常是预览渲染或缓存问题；如果点开也空白，通常是页面图片抽取、裁剪边界或视觉结构识别问题。安装 Poppler 后重新解析通常能改善 PDF 页面图片和裁剪质量。

## 8. 导出笔记

读完后点：

- “下载笔记”：导出 Markdown，适合 Obsidian、Notion 或继续手工整理。
- “下载Word”：导出 `.docx`，适合打印、分享或继续编辑。

导出前建议先点“导出检查”。如果有缺翻译、缺讲解、低置信公式或图表引用异常，先补跑或修复，再导出。

## 9. 长任务怎么处理

整篇论文精读可能要很久，尤其是页数多、公式多、图多的时候。使用时记住三点：

1. 刷新页面不会让后端任务立刻消失，任务会在后端队列里继续记录。
2. 失败段落可以补跑，不需要每次从头跑。
3. 分段整体错才需要重分段，普通失败优先补跑失败/未完成。

如果服务真的断了，先运行：

```bash
npm run service:restart
npm run health
```

再回页面看任务历史和质量报告。

## 10. 迁移和备份

换电脑或升级前，导出数据包：

```bash
npm run data:export
```

在新电脑导入：

```bash
npm run data:import -- /path/to/paperlens-data-时间戳.tar.gz --yes
```

默认不会带 `.env` 和 `data/secrets.json`。这能避免你把 API Key 意外发给别人。
