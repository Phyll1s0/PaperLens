# PaperLens Roadmap

PaperLens 接下来优先从“能跑”走向“长期可靠地跑”。路线按地基到体验排序，避免先做华丽功能却被长任务、密钥和 PDF 结构拖住。

## 1. Job Queue v2

目标：长任务可恢复、可观察、可重试。

- 后端持久化分析任务已经落地，前端只创建任务和轮询状态。
- 分析任务已支持批量调用模型，降低 Claude Code Agent 每段启动一次的固定开销。
- 页面 Kimi Code Key 默认走 Kimi Code Direct，直连 Anthropic-compatible endpoint，避免本机 `claude` CLI 启动慢、预算串号和 Docker 缺 CLI 的问题。
- TimesFM 论文已完成一次整篇真实 smoke test：`125/125` 段完成，失败 `0`，导出 QA `ok`。
- 整篇分析模式已改为质量优先批处理：保留上下文窗口，讲解覆盖含义、作用、关键难点和图表/公式关系，并发跑多个批次；大批次失败时拆成小批次重试，避免回退到超慢单段路径。
- 继续补强任务历史、失败项重试、只重跑失败段落、失败原因展示。
- 之后加入更细的重试退避、并发限制和任务清理策略。

## 2. API Key 安全

目标：后端任务能断开前端继续跑，同时不把完整密钥散落在任务文件里。

- Job 中不直接保存完整 API Key。
- 已增加本地私有 key store，页面和 Job 都只保存 key id。
- 完整 Key 不再写入浏览器 `sessionStorage`，后端保存文件使用本机私有 `data/secrets.json`。
- 页面继续保持本地优先；云端部署时明确要求用户自管环境变量或本地存储。

## 3. PDF 视觉结构

目标：图、表、公式、代码都成为可引用的结构化对象。

- 当前图表裁剪基于版面几何和文字块启发式。
- 已为公式块和代码块加入页面截图裁剪与点击放大。
- 下一步继续细化纯图片边界和跨栏图表裁剪。
- 图表 artifact 继续支持点击放大、整页定位、段落引用。

## 4. AI 分段与上下文

目标：每段讲解不孤立，能理解前文和章节脉络。

- 分析单段时已加入前后段落、章节标题、相关图表 caption 和前文术语。
- AI 分段已加入上一窗口摘要、尾段和关键词，减少跨 chunk 断裂。
- AI 分段已加入全文结构预扫描，先识别正文起点、References 起点、章节边界和非正文区域，再按窗口分段。
- PDF 视觉结构已加入 `visualRegions`，用于更精细裁剪图片/表格/公式/代码，并把图内文字从正文候选里过滤出去。
- 段落分析已加入全文关键词、章节摘要和同引用窗口。
- 后续用检索替代固定邻近窗口。

## 5. 分发与开源体验

目标：别人能明确、稳定、低成本地跑起来。

- README 拆清本机、Docker、代理、Claude Code、OpenAI-compatible 配置。
- 提供一键启动和健康检查路径。
- 后续再评估 Electron/Tauri 桌面端或轻量 Web 部署模板。

## 6. AI-first Layout

目标：让 AI 参与全文版面理解，同时保留本地校验、裁剪和回退。

- 第一阶段先定义 AI layout JSON schema，把标题、章节、段落、阅读顺序、图表/公式/代码区域统一成可校验结构。
- 第二阶段做 mock/command provider，让模型或外部脚本输出 layout JSON。
- 第三阶段做 hybrid 模式：本地 PDF 抽取仍保留，AI 负责修正文起点、章节、跨页段落和视觉区域。
- 第四阶段再接支持 PDF/vision 的真实模型，并把耗时、费用、失败率放进质量报告和任务估算。

## 当前执行顺序

1. 自动化测试基础层：PDF fixture、Provider mock、Kimi Direct mock、碎片 LaTeX、导出 QA 和 Job Queue 恢复。
2. 长任务预算保护：分析前展示预计 token/时间/额度，支持任务级最大预算。
3. AI-first Layout schema：先完成本地校验协议和 mock provider，再进入真实模型接入。
4. PDF 视觉结构：继续细化纯图片边界、跨栏图表裁剪，并进入可手动修正的视觉裁剪编辑器。
